## Пометки (что где и зачем)

- `drivers/` — только команды приборов (SCPI/ASCII/Serial). Меняем тут, если меняются команды/тайминги.
- `transfer.py` — процедура реверса (+/-) и логирование. Меняем тут, если меняется методика.
- `math/` — расчёты (дельта, неопределённость, приведение к 20°C). Меняем тут, если меняются формулы.
- `io/` — форматы сохранения (CSV/XLSX). Меняем тут, если меняется формат отчёта/логов.
- `service/server.py` — HTTP сервис на RPi (выполняет измерения).
- `client/win_gui.py` — приложение Windows (только управление и отображение).

### Где задаётся температура
- LTE-300 читается **на каждое измерение напряжения** в `transfer.py` (функция `_acquire_block_with_lte`).
- BME280 читается фоновым потоком в `service/server.py` и показывается в GUI через `/env`.

### Приведение к 20°C
- `math/ne_temp.py`
  - насыщенный: полином (коэффициенты 20°C)
  - ненасыщенный: альфа из паспорта (мкВ/°C)

## Добавлено в v0.0.2: LTE-300 по /dev/serial/by-id (или по SerialNumber)

Почему:
- `/dev/ttyUSB0` может меняться после перезагрузки.
- `/dev/serial/by-id/...` стабилен и привязан к конкретному USB-UART (FTDI).

Как использовать:
1) Полный путь:
   - `/dev/serial/by-id/usb-FTDI_UT232R_FT4QUE2F-if00-port0`

2) Упрощённо по серийному номеру (substring):
   - `by-id:FT4QUE2F`

Где реализовано:
- `transfer_service/drivers/lte300.py`:
  - функция `resolve_lte_port()`
  - `LTE300Config.port` принимает path или `by-id:...`

## Зафиксировано в v0.0.6: режим Simulation (эмуляция 2182 + 320B + LTE-300)

Как включить:
- В GUI поставить галочку **Enable simulation**
  или
- В запросе `/start` поставить `"simulation": true`
  (также можно указать ресурсы `meter_resource="MOCK"`, `switch_resource="MOCK"`)

Что эмулируется:
- 2182: сигнал `offset` (µV) + шум (µV RMS) + дрейф (µV/min) + редкие выбросы
- 320B: хранит последнее состояние A/B и задаёт полярность (+/-) для эмулятора 2182
- LTE-300: температура `sim_temp_c` + небольшой шум

Параметры:
- `sim_offset_uV`, `sim_noise_rms_uV`, `sim_drift_uV_per_min`, `sim_outlier_prob`, `sim_outlier_uV`, `sim_temp_c`

## Зафиксировано в v0.0.8: сбор по количеству + исправление математики

Изменения (по итогам обсуждения):
- Убрали сбор "по времени" (`block_duration_s`) в пользу фиксированного количества точек.
  - Новый параметр: `samples_per_polarity` — число измерений на каждую полярность в одном цикле.
  - Теперь всегда: `n_plus == n_minus == cycles * samples_per_polarity`.
  - Поле `block_duration_s` оставлено как legacy (<= v0.0.7), в v0.0.8 не используется (в UI отправляется 0).
- Исправлена математика Type A и неопределённости:
  - Type A считается по массиву `delta_i = (plus_i - minus_i)/2` (а не по объединённым `plus+minus`).
  - `estimate_uncertainty()` теперь принимает `delta_samples` и комбинирует `u_typeA` с `u_ref`.
- Исправлены ошибки запуска и зависимостей:
  - Добавлен `pyserial` в `requirements.txt` (нужен драйверу LTE-300).
  - Сохранён фикс синтаксиса `_worker` в `server.py` из v0.0.7.1 (ошибка отступов).

Файлы, которые трогали:
- `transfer_service/transfer.py` — MeasurePlan + сбор по количеству + корректная статистика.
- `math/uncertainty.py` — корректная оценка по `delta_samples`.
- `service/server.py` — добавлен `samples_per_polarity` в StartRequest и передача в MeasurePlan.
- Web UI: `templates/index.html`, `static/app.js` — поле `samples / polarity`.
